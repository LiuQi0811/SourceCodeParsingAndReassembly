API 文档

Web Locks API 完整文档

核心对象

OBJECT navigator.locks

LockManager 对象，提供锁请求和查询方法。

主要方法

METHOD locks.request(name, options, callback)

请求锁资源，在回调中执行需要互斥访问的操作。

METHOD locks.query()

返回当前锁状态信息，包括持有和等待中的锁。

LockRequestOptions

属性类型说明
modestring锁模式: "exclusive" 或 "shared"
ifAvailableboolean是否立即获取或失败
stealboolean是否抢占锁
signalAbortSignal用于取消请求

Lock 对象属性

属性说明
name锁名称
mode锁模式
held是否被持有
clientId持有者客户端ID

使用示例

// 请求独占锁 await navigator.locks.request('my-resource', async (lock) => { // 在锁保护下执行操作 console.log('持有锁:', lock.name); await doSomething(); // 回调结束时自动释放锁 }); // 请求共享锁 await navigator.locks.request('my-resource', { mode: 'shared' }, async (lock) => { await readData(); }); // 立即获取或失败 const result = await navigator.locks.request('my-resource', { ifAvailable: true }, async (lock) => { if (!lock) { console.log('锁不可用'); return null; } return await doWork(); }); // 查询锁状态 const state = await navigator.locks.query(); console.log('持有的锁:', state.held); console.log('等待中的锁:', state.pending);
