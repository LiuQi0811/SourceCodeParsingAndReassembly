Channel Messaging API 概览

Channel Messaging API 提供了一种在两个浏览上下文之间建立双向通信通道的方法

📡

MessageChannel

创建新的消息通道，返回两个相互连接的 MessagePort

🔌

MessagePort

消息端口，用于发送和接收消息

📦

可转移对象

支持转移所有权的高效数据传输

🔗

双向通信

两个端口可以相互发送和接收消息

📊 API 支持检测

API 状态 说明

⚡ 快速开始

// 创建消息通道 const channel = new MessageChannel(); // 获取两个端口 const port1 = channel.port1; const port2 = channel.port2; // 设置消息监听 port1.addEventListener('message', (e) => { console.log('Port1 收到:', e.data); }); port1.start(); port2.addEventListener('message', (e) => { console.log('Port2 收到:', e.data); }); port2.start(); // 发送消息 port1.postMessage('Hello from Port1!'); port2.postMessage('Hello from Port2!');

MessagePort 接口

MessagePort 接口表示消息通道的一个端口，可以发送和接收消息

🔧 方法

方法 参数 说明
postMessage() message, [transfer] 通过通道发送消息
start() 无 开始分发消息（使用 addEventListener 时需要调用）
close() 无 断开端口连接，不再接收消息

📡 事件

onmessage Event
收到消息时触发
onmessageerror Event
消息反序列化失败时触发

属性 类型 说明

originstring消息来源的源
lastEventIdstring最后一个事件ID
sourceWindow/MessagePort消息来源对象
portsMessagePort[]随消息发送的端口数组

👷 Worker 双向通信演示

ℹ️
Worker 通信原理
通过将 MessagePort 转移给 Worker，实现主线程与 Worker 的直接双向通信

Worker

📝 Worker 代码示例

// 主线程代码 const worker = new Worker('worker.js'); const channel = new MessageChannel(); // 将 port2 转移给 Worker worker.postMessage({ type: 'init' }, [channel.port2]); // 使用 port1 与 Worker 通信 channel.port1.postMessage({ task: 'process' }); channel.port1.addEventListener('message', (e) => { console.log('Worker 响应:', e.data); }); channel.port1.start(); // worker.js self.addEventListener('message', (e) => { if (e.data.type === 'init') { const port = e.ports[0]; port.addEventListener('message', (event) => { port.postMessage({ result: 'done' }); }); port.start(); } });

📢 BroadcastChannel 演示

ℹ️
BroadcastChannel API
BroadcastChannel 允许同源的不同浏览器上下文（如不同标签页、窗口、iframe）之间进行通信

📝 BroadcastChannel 代码示例

// 创建广播频道 const bc = new BroadcastChannel('my-channel'); // 监听消息 bc.addEventListener('message', (e) => { console.log('收到广播:', e.data); }); // 发送广播 bc.postMessage({ type: 'greeting', text: 'Hello!' }); // 关闭频道 bc.close();
