Credential Management API 概览

Credential Management API 提供了管理用户认证凭据的能力

📥

获取凭据

使用 get() 方法获取存储的凭据

💾

存储凭据

使用 store() 方法保存凭据

🆕

创建凭据

使用 create() 方法创建新凭据

🔒

防止静默访问

使用 preventSilentAccess() 控制自动登录

📊 API 支持检测

API 状态 说明

⚡ 快速开始

// 检查 API 支持 if (navigator.credentials) { // 获取凭据 const credential = await navigator.credentials.get({ password: true, federated: { providers: ['https://accounts.google.com'] } }); // 存储凭据 await navigator.credentials.store( new PasswordCredential({ id: 'user@example.com', password: 'secret' }) ); // 防止静默访问 await navigator.credentials.preventSilentAccess(); }

⚠️ 使用要求

⚠️
安全要求
Credential Management API 需要以下条件：
  • 必须通过 HTTPS 提供页面（localhost 除外）
  • 某些操作需要用户手势触发
  • 需要用户授权

CredentialsContainer 接口

CredentialsContainer 提供凭据管理的核心方法

📍 访问方式

navigator.credentials CredentialsContainer
全局可访问

🔧 方法

方法 参数 返回值 说明
get() CredentialRequestOptions Promise<Credential?> 获取凭据
store() Credential Promise<Credential> 存储凭据
create() CredentialCreationOptions Promise<Credential?> 创建凭据
preventSilentAccess() 无 Promise<void> 防止静默访问

Credential 接口

Credential 是所有凭据类型的基类

📋 属性

属性 类型 说明
id string 凭据标识符
type string 凭据类型

🏷️ 凭据类型

类型 说明
password密码凭据
federated联合身份凭据
public-key公钥凭据 (WebAuthn)

PasswordCredential 接口

PasswordCredential 表示用户名/密码凭据

🏗️ 构造函数

new PasswordCredential(data) // data 可以是: // 1. HTMLFormElement - 表单元素 // 2. PasswordCredentialData - 数据对象 const credential = new PasswordCredential({ id: 'user@example.com', password: 'secret123', name: 'John Doe', iconURL: 'https://example.com/avatar.png' });

📋 属性

属性 类型 说明
idstring用户名/邮箱
passwordstring密码
namestring用户显示名称
iconURLstring用户头像 URL

FederatedCredential 接口

FederatedCredential 表示联合身份提供者的凭据

🏗️ 构造函数

const credential = new FederatedCredential({ id: 'user@example.com', name: 'John Doe', iconURL: 'https://example.com/avatar.png', provider: 'https://accounts.google.com', protocol: 'openidconnect' });

📋 属性

属性 类型 说明
idstring用户 ID
namestring用户显示名称
iconURLstring用户头像 URL
providerstring身份提供者
protocolstring协议

🌐 常见提供者

Google https://accounts.google.com
Facebook https://www.facebook.com
GitHub https://github.com
Twitter https://twitter.com
