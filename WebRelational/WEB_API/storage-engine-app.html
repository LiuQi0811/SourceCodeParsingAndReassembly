API文档

存储引擎完整API接口文档

对象操作接口

GET /api/v1/objects/{bucket}/{key}

下载对象（文件）

路径参数

bucket string 存储桶名称
key string 对象键（文件路径）

查询参数

versionId string 指定版本ID（启用版本控制时）
responseContentType string 覆盖响应Content-Type
// 响应示例 HTTP/1.1 200 OK Content-Type: image/jpeg Content-Length: 12345 ETag: "d41d8cd98f00b204e9800998ecf8427e" Last-Modified: Mon, 15 Jan 2024 10:30:00 GMT [binary data]
PUT /api/v1/objects/{bucket}/{key}

上传对象（文件）

请求头

Content-Type string 文件MIME类型
Content-MD5 string 内容MD5校验值
x-amz-storage-class string 存储类型: STANDARD|REDUCED_REDUNDANCY|GLACIER
// 请求示例 PUT /api/v1/objects/my-bucket/images/photo.jpg HTTP/1.1 Content-Type: image/jpeg Content-Length: 12345 [binary data] // 响应示例 { "etag": "\"d41d8cd98f00b204e9800998ecf8427e\"", "versionId": "v1", "location": "/my-bucket/images/photo.jpg" }
DELETE /api/v1/objects/{bucket}/{key}

删除对象

GET /api/v1/objects/{bucket}/{key}/metadata

获取对象元数据（HEAD请求）

PUT /api/v1/objects/{bucket}/{key}/copy

复制对象

// 请求示例 PUT /api/v1/objects/dest-bucket/dest-key/copy HTTP/1.1 Content-Type: application/json { "sourceBucket": "source-bucket", "sourceKey": "source-key", "metadataDirective": "COPY" }
POST /api/v1/objects/{bucket}/delete-batch

批量删除对象

{ "objects": [ {"key": "file1.jpg"}, {"key": "file2.jpg"}, {"key": "folder/file3.pdf"} ], "quiet": false }

存储桶操作接口

GET /api/v1/buckets

列出所有存储桶

{ "buckets": [ { "name": "my-bucket", "creationDate": "2024-01-15T10:30:00Z", "region": "cn-north" } ], "owner": { "id": "user-123", "displayName": "Admin" } }
POST /api/v1/buckets

创建存储桶

{ "name": "my-new-bucket", "region": "cn-north", "storageClass": "STANDARD", "acl": "private", "versioning": false }
DELETE /api/v1/buckets/{bucket}

删除存储桶（需先清空）

GET /api/v1/buckets/{bucket}/objects

列出存储桶中的对象

查询参数

prefix string 对象键前缀筛选
delimiter string 分隔符，用于模拟文件夹
maxKeys integer 最大返回数量，默认1000
marker string 分页标记
GET /api/v1/buckets/{bucket}/stats

获取存储桶统计信息

分片上传接口

POST /api/v1/multipart/{bucket}/{key}/initiate

初始化分片上传

// 响应示例 { "uploadId": "YzQ...abc123", "bucket": "my-bucket", "key": "large-file.zip" }
PUT /api/v1/multipart/{bucket}/{key}/parts/{uploadId}/{partNumber}

上传分片

路径参数

uploadId string 上传任务ID
partNumber integer 分片序号(1-10000)
GET /api/v1/multipart/{bucket}/{key}/parts/{uploadId}

列出已上传的分片

POST /api/v1/multipart/{bucket}/{key}/complete

完成分片上传

{ "uploadId": "YzQ...abc123", "parts": [ {"partNumber": 1, "etag": "\"abc123\""}, {"partNumber": 2, "etag": "\"def456\""}, {"partNumber": 3, "etag": "\"ghi789\""} ] }
DELETE /api/v1/multipart/{bucket}/{key}/abort/{uploadId}

取消分片上传

权限管理接口

GET /api/v1/acl/{bucket}

获取存储桶ACL

GET /api/v1/acl/{bucket}/{key}

获取对象ACL

PUT /api/v1/acl/{bucket}

设置存储桶ACL

{ "owner": { "id": "user-123" }, "grants": [ { "grantee": { "type": "CanonicalUser", "id": "user-456" }, "permission": "READ" }, { "grantee": { "type": "Group", "uri": "http://acs.amazonaws.com/groups/global/AllUsers" }, "permission": "READ" } ] }
PUT /api/v1/acl/{bucket}/{key}

设置对象ACL

GET /api/v1/policy/{bucket}

获取存储桶策略

PUT /api/v1/policy/{bucket}

设置存储桶策略

签名URL接口

POST /api/v1/signed-url

生成预签名URL

{ "bucket": "my-bucket", "key": "images/photo.jpg", "method": "GET", "expires": 3600, "contentType": "image/jpeg" } // 响应示例 { "url": "https://storage.example.com/my-bucket/images/photo.jpg?X-Amz-Algorithm=...", "expiresAt": "2024-01-15T12:00:00Z" }
POST /api/v1/signed-url/batch

批量生成预签名URL

{ "bucket": "my-bucket", "keys": ["file1.jpg", "file2.jpg", "file3.pdf"], "method": "GET", "expires": 3600 }
