API Introduction

The Cooperative Scheduling of Background Tasks API (also referred to as the Background Tasks API or the requestIdleCallback() API) provides the ability to queue tasks to be executed automatically by the user agent when it determines that there is free time to do so.

🎯 Core Features

  • ✓ Execute tasks during browser idle periods
  • ✓ Non-blocking background work on main thread
  • ✓ No impact on latency-critical events
  • ✓ Timeout support for guaranteed execution
  • ✓ Cancel scheduled callbacks
  • ✓ Query remaining idle time

📦 API Components

requestIdleCallback() Method
cancelIdleCallback() Method
IdleDeadline Interface
IdleDeadline.timeRemaining() Method
IdleDeadline.didTimeout Property

💻 Quick Start

// Schedule a task to run during idle periods const taskId = requestIdleCallback((deadline) => { // Check if we're running due to timeout if (deadline.didTimeout) { console.log('Callback ran due to timeout'); } // Process tasks while we have time while (deadline.timeRemaining() > 0 && hasMoreWork()) { processNextTask(); } // If there's more work, schedule another callback if (hasMoreWork()) { requestIdleCallback(processTasks); } }, { timeout: 2000 }); // Cancel the callback if needed cancelIdleCallback(taskId);

▶️ requestIdleCallback()

Method

The window.requestIdleCallback() method queues a function to be called during a browser's idle periods. This enables developers to perform background and low priority work on the main thread, without impacting latency-critical events such as animation and input response.

Syntax

const id = requestIdleCallback(callback); const id = requestIdleCallback(callback, options);

📥 Parameters

Parameter Type Description
callback Function Function to be called during idle periods. Receives IdleDeadline object.
options.timeout number Maximum time in ms before callback is executed regardless of idle state.

📤 Return Value

id unsigned long

An ID which can be used to cancel the callback by passing it into the window.cancelIdleCallback() method.

💻 Examples

// Basic usage requestIdleCallback((deadline) => { console.log('Idle time remaining:', deadline.timeRemaining()); }); // With timeout requestIdleCallback((deadline) => { if (deadline.didTimeout) { console.log('Callback executed due to timeout'); } // Do work... }, { timeout: 1000 }); // Cooperative scheduling pattern function processTasks(deadline) { while (deadline.timeRemaining() > 0 && tasks.length > 0) { const task = tasks.shift(); processTask(task); } if (tasks.length > 0) { requestIdleCallback(processTasks); } } requestIdleCallback(processTasks);

⏹️ cancelIdleCallback()

Method

The window.cancelIdleCallback() method cancels a callback previously scheduled with window.requestIdleCallback().

Syntax

cancelIdleCallback(id);

📥 Parameters

Parameter Type Description
id unsigned long The ID returned by requestIdleCallback()

📤 Return Value

undefined void

💻 Examples

// Schedule a callback const taskId = requestIdleCallback(() => { console.log('Running idle callback'); }); // Cancel before it runs cancelIdleCallback(taskId); // Practical example: Cancel on user interaction let pendingTaskId = null; function scheduleBackgroundWork() { // Cancel any pending work if (pendingTaskId !== null) { cancelIdleCallback(pendingTaskId); } // Schedule new work pendingTaskId = requestIdleCallback((deadline) => { pendingTaskId = null; doBackgroundWork(deadline); }); }

⏱️ IdleDeadline Interface

Interface

The IdleDeadline interface is used as the data type of the input parameter to idle callbacks established by calling Window.requestIdleCallback(). It offers a method, timeRemaining(), which lets you determine how much longer the user agent estimates it will remain idle and a property, didTimeout, which lets you determine if your callback is executing because its timeout duration expired.

Web IDL Definition

interface IdleDeadline { readonly attribute boolean didTimeout; DOMHighResTimeStamp timeRemaining(); };

⚠️ didTimeout Property

Read-only

A boolean value that is true if the callback is being executed because the timeout specified when the idle callback was registered has expired.

Type boolean

⏳ timeRemaining() Method

Method

Returns a DOMHighResTimeStamp value (a floating-point number) representing the amount of time remaining before the idle period ends.

Return Type DOMHighResTimeStamp

💻 Examples

// Using IdleDeadline properties requestIdleCallback((deadline) => { // Check if we ran due to timeout if (deadline.didTimeout) { console.log('Callback executed due to timeout!'); } // Check remaining time const remaining = deadline.timeRemaining(); console.log(`Time remaining: ${remaining}ms`); // Process tasks cooperatively while (deadline.timeRemaining() > 5 && hasWork()) { processWorkItem(); } }); // Complete example: Task queue processor const taskQueue = []; function processQueue(deadline) { // Process as many tasks as possible in remaining time while (deadline.timeRemaining() > 0 && taskQueue.length > 0) { const task = taskQueue.shift(); task(); } // If there are more tasks, schedule again if (taskQueue.length > 0) { requestIdleCallback(processQueue); } } // Start processing requestIdleCallback(processQueue);
