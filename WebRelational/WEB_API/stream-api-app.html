Reader & Writer

流读取器和写入器管理

ReadableStreamDefaultReader

METHOD reader.read()

从流中读取下一个数据块。返回 Promise<{done: boolean, value: any}>

METHOD reader.cancel(reason?)

取消读取操作，关闭流。

METHOD reader.releaseLock()

释放对流的锁定，允许其他 reader 获取流。

WritableStreamDefaultWriter

METHOD writer.write(chunk)

向流写入一个数据块。返回 Promise。

METHOD writer.close()

关闭流，表示写入完成。

METHOD writer.abort(reason?)

中止流，丢弃所有排队的数据。

PROPERTY writer.desiredSize

返回期望写入的队列大小，用于背压控制。

PROPERTY writer.ready

返回 Promise，当流准备好接受数据时 resolve。

流锁定状态

流ID 类型 锁定状态 锁定者 操作

字节流

Bytes Stream - 二进制数据处理

CONSTRUCTOR new ReadableStream({ type: 'bytes' })

创建一个字节流，支持高效的二进制数据处理。

const byteStream = new ReadableStream({ type: 'bytes', autoAllocateChunkSize: 1024, start(controller) { // 初始化 }, pull(controller) { // 按需拉取数据 const buffer = new ArrayBuffer(1024); controller.enqueue(new Uint8Array(buffer)); } });
PROPERTY controller.byobRequest

获取 BYOB (Bring Your Own Buffer) 请求对象，用于零拷贝读取。

ReadableByteStreamController

METHOD controller.enqueue(chunk)

将 Uint8Array 数据块加入队列。

METHOD controller.close()

关闭流。

METHOD controller.error(e)

发出错误信号。

PROPERTY controller.byobRequest

返回当前的 BYOB 请求，或 null。

PROPERTY controller.desiredSize

返回期望的队列大小。

API 文档

Web Streams API 完整接口文档

ReadableStream API

CONSTRUCTOR new ReadableStream(underlyingSource?, queuingStrategy?)

创建一个新的可读流。

underlyingSource 参数

start(controller) function 流创建时立即调用
pull(controller) function 当内部队列不满时重复调用
cancel(reason) function 流被取消时调用
type string 'bytes' 或 undefined
const readable = new ReadableStream({ start(controller) { controller.enqueue('Hello'); controller.enqueue('World'); controller.close(); } });
METHOD readable.getReader(options?)

获取流的读取器，锁定流。

Options

mode string 'byob' 用于字节流
METHOD readable.pipeTo(destination, options?)

将流管道传输到可写流。

METHOD readable.pipeThrough(transform, options?)

通过转换流管道传输。

METHOD readable.tee()

将流分成两个独立的流。

const [stream1, stream2] = readable.tee(); // stream1 和 stream2 包含相同的数据
METHOD readable.cancel(reason?)

取消流。

PROPERTY readable.locked

返回流是否被锁定。

WritableStream API

CONSTRUCTOR new WritableStream(underlyingSink?, queuingStrategy?)

创建一个新的可写流。

underlyingSink 参数

start(controller) function 流创建时调用
write(chunk, controller) function 写入数据块时调用
close(controller) function 流关闭时调用
abort(reason) function 流中止时调用
const writable = new WritableStream({ write(chunk) { console.log('Received:', chunk); }, close() { console.log('Stream closed'); } });
METHOD writable.getWriter()

获取流的写入器。

METHOD writable.abort(reason?)

中止流。

METHOD writable.close()

关闭流。

PROPERTY writable.locked

返回流是否被锁定。

TransformStream API

CONSTRUCTOR new TransformStream(transformer?, writableStrategy?, readableStrategy?)

创建一个新的转换流。

transformer 参数

start(controller) function 转换开始时调用
transform(chunk, controller) function 处理每个数据块
flush(controller) function 流结束时调用
const upperCaseTransform = new TransformStream({ transform(chunk, controller) { controller.enqueue(chunk.toUpperCase()); } });
PROPERTY transform.readable

返回可读端。

PROPERTY transform.writable

返回可写端。

静态方法

STATIC ReadableStream.from(asyncIterable)

从异步可迭代对象创建可读流。

// 从数组创建 const stream = ReadableStream.from([1, 2, 3, 4, 5]); // 从异步生成器创建 async function* generate() { yield 'Hello'; yield 'World'; } const stream2 = ReadableStream.from(generate());
STATIC ByteLengthQueuingStrategy

基于字节长度的队列策略。

const strategy = new ByteLengthQueuingStrategy({ highWaterMark: 1024 * 1024 // 1MB });
STATIC CountQueuingStrategy

基于数据块数量的队列策略。

const strategy = new CountQueuingStrategy({ highWaterMark: 10 // 最多10个数据块 });
