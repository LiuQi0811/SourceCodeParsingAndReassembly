Contact Picker API 概览

Contact Picker API 允许 Web 应用访问用户设备上的联系人信息

📱

选择联系人

使用 select() 方法打开系统联系人选择器

📋

获取属性

使用 getProperties() 获取设备支持的联系人属性

🔐

隐私保护

用户主动选择，仅返回请求的属性

✅

多选支持

支持单选或多选联系人

📊 API 支持检测

API 状态 说明

⚡ 快速开始

// 检查 API 支持 if ('contacts' in navigator && 'select' in navigator.contacts) { // 获取支持的属性 const props = await navigator.contacts.getProperties(); // 选择联系人 const contacts = await navigator.contacts.select( ['name', 'email', 'tel'], { multiple: true } ); // 处理联系人数据 contacts.forEach(contact => { console.log(contact.name, contact.email, contact.tel); }); }

⚠️ 使用要求

⚠️
安全要求
Contact Picker API 需要以下条件才能使用：
  • 必须通过 HTTPS 提供页面
  • 必须由用户手势触发（如点击）
  • 必须在顶级浏览上下文中使用
  • 主要支持移动端浏览器

navigator.contacts 接口

ContactsManager 接口提供访问设备联系人的能力

📍 访问方式

navigator.contacts ContactsManager
通过 navigator 对象访问

🔧 方法

方法 参数 返回值 说明
select() properties[], options? Promise<ContactInfo[]> 打开联系人选择器
getProperties() 无 Promise<ContactProperty[]> 获取支持的属性列表

ContactInfo 接口

ContactInfo 表示单个联系人的信息

📋 属性

属性 类型 说明
name string[] 联系人姓名数组
email string[] 邮箱地址数组
tel string[] 电话号码数组
address ContactAddress[] 地址数组
icon Blob[] 头像图片数组
