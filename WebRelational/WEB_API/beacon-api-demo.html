API Introduction

The Beacon API is used to send an asynchronous and non-blocking request to a web server. The request does not expect a response. Unlike requests made using XMLHttpRequest or the Fetch API, the browser guarantees to initiate beacon requests before the page is unloaded and to run them to completion.

🎯 Core Features

  • ✓ Asynchronous and non-blocking
  • ✓ Guaranteed delivery on page unload
  • ✓ No response expected
  • ✓ Low overhead for analytics
  • ✓ Works with various data types
  • ✓ Better than synchronous XHR

📦 API Components

navigator.sendBeacon() Method

💻 Quick Start

// Send analytics data const data = { event: 'page_view', timestamp: Date.now(), url: window.location.href }; // Send beacon const success = navigator.sendBeacon( '/analytics', JSON.stringify(data) ); if (success) { console.log('Beacon queued successfully'); } else { console.log('Beacon failed to queue'); }

▶️ navigator.sendBeacon()

Method

The navigator.sendBeacon() method asynchronously sends an HTTP POST request containing a small amount of data to a web server. It's intended for analytics and diagnostics code that needs to send data before the page unloads.

Syntax

const success = navigator.sendBeacon(url); const success = navigator.sendBeacon(url, data);

📥 Parameters

Parameter Type Description
url string The URL to send the beacon to
data optional Data to send (string, Blob, FormData, etc.)

📤 Return Value

boolean true | false

Returns true if the browser successfully queued the beacon request for delivery, false otherwise.

💻 Examples

// Basic usage - URL only navigator.sendBeacon('/analytics/ping'); // Send string data navigator.sendBeacon('/analytics', 'event=click&element=button'); // Send JSON data const analyticsData = JSON.stringify({ event: 'page_view', page: window.location.pathname, timestamp: Date.now() }); navigator.sendBeacon('/analytics', analyticsData); // Send FormData const formData = new FormData(); formData.append('event', 'submit'); formData.append('form_id', 'contact'); navigator.sendBeacon('/analytics', formData); // Send Blob with content type const blob = new Blob([JSON.stringify({ event: 'click' })], { type: 'application/json' }); navigator.sendBeacon('/analytics', blob); // On page unload window.addEventListener('unload', () => { navigator.sendBeacon('/analytics/session-end', JSON.stringify({ duration: performance.now(), page: window.location.href })); });

Supported Data Types

The sendBeacon() method accepts various data types for the second parameter. The browser will set the appropriate Content-Type header based on the data type.

📦 Supported Data Types

Type Content-Type Description
string text/plain;charset=UTF-8 Plain text or URL-encoded string
ArrayBuffer application/octet-stream Binary data buffer
TypedArray application/octet-stream Typed array view (Uint8Array, etc.)
DataView application/octet-stream Data view of ArrayBuffer
Blob blob.type Binary large object with custom type
FormData multipart/form-data Form data object
URLSearchParams application/x-www-form-urlencoded URL query parameters
ReadableStream application/octet-stream Stream of data

💻 Examples by Data Type

// String navigator.sendBeacon('/log', 'Hello World'); // ArrayBuffer const buffer = new ArrayBuffer(8); const view = new DataView(buffer); view.setFloat64(0, Math.PI); navigator.sendBeacon('/log', buffer); // TypedArray (Uint8Array) const uint8 = new Uint8Array([1, 2, 3, 4, 5]); navigator.sendBeacon('/log', uint8); // Blob with JSON content type const jsonBlob = new Blob( [JSON.stringify({ event: 'click', x: 100, y: 200 })], { type: 'application/json' } ); navigator.sendBeacon('/analytics', jsonBlob); // FormData const formData = new FormData(); formData.append('username', 'john'); formData.append('action', 'login'); navigator.sendBeacon('/log', formData); // URLSearchParams const params = new URLSearchParams({ event: 'pageview', page: '/home', time: Date.now() }); navigator.sendBeacon('/analytics', params);
