Experimental API

Background Synchronization API enables web applications to defer actions until the user has stable connectivity. This is ideal for offline-first applications that need to sync data when connectivity is restored.

🎯 Core Features

  • ✓ Defer tasks until network connectivity is available
  • ✓ Works even when the app is closed
  • ✓ Retry mechanism for failed syncs
  • ✓ Unique tag-based identification
  • ✓ Service Worker integration
  • ✓ Periodic sync for regular updates

📦 SyncManager Interface

Experimental

The SyncManager interface provides methods for registering and listing sync registrations. It is accessed through ServiceWorkerRegistration.sync.

Web IDL Definition

[Exposed=(Window,Worker)] interface SyncManager { Promise<void> register(DOMString tag); Promise<sequence<DOMString>> getTags(); };

▶️ register() Method

Method

Registers a sync event with the given tag. The sync event will be fired in the Service Worker when the user has connectivity.

Parameters

tag DOMString - Unique identifier for the sync event

Return Value

Promise<void> Promise

Exceptions

InvalidStateError No active Service Worker
NotAllowedError Permission denied

📋 getTags() Method

Method

Returns a list of all registered sync tags.

Return Value

Promise<sequence<DOMString>> Promise

💻 Usage Example

// Register a background sync async function registerSync() { try { const registration = await navigator.serviceWorker.ready; await registration.sync.register('my-sync-tag'); console.log('Sync registered!'); } catch (error) { console.error('Sync registration failed:', error); } } // Get all registered sync tags async function getSyncTags() { const registration = await navigator.serviceWorker.ready; const tags = await registration.sync.getTags(); console.log('Registered tags:', tags); }

⚡ SyncEvent Interface

Experimental

The SyncEvent interface represents a sync action dispatched on the ServiceWorkerGlobalScope. It extends the ExtendableEvent interface.

Web IDL Definition

[Exposed=ServiceWorker] interface SyncEvent : ExtendableEvent { readonly attribute DOMString tag; readonly attribute boolean lastChance; };

🏷️ tag Property

Read-only

Returns the developer-defined identifier for this SyncEvent.

Type DOMString

⚠️ lastChance Property

Read-only

Returns true if the user agent will not make further synchronization attempts after the current attempt.

Type boolean

🔧 SyncEvent Constructor

new SyncEvent(type, options); // Parameters: // type: DOMString - The event type (typically "sync") // options: SyncEventInit dictionary dictionary SyncEventInit : ExtendableEventInit { required DOMString tag; boolean lastChance = false; };

📡 Service Worker Event Handler

// In service-worker.js self.addEventListener('sync', (event) => { console.log('Sync event received:', event.tag); console.log('Is last chance:', event.lastChance); if (event.tag === 'sync-messages') { event.waitUntil( syncMessages().catch((error) => { if (event.lastChance) { console.error('Final sync attempt failed'); } throw error; }) ); } });

Periodic Background Synchronization

The Periodic Background Synchronization API allows web applications to register tasks to be run in a service worker at periodic intervals with network connectivity.

🕐 PeriodicSyncManager Interface

Experimental

The PeriodicSyncManager interface provides methods for registering and managing periodic sync events.

Web IDL Definition

[Exposed=(Window,Worker)] interface PeriodicSyncManager { Promise<void> register(DOMString tag, optional PeriodicSyncOptions options = {}); Promise<sequence<DOMString>> getTags(); Promise<void> unregister(DOMString tag); }; dictionary PeriodicSyncOptions { BackgroundSyncPeriodicSyncInterval minInterval; }; typedef unsigned long BackgroundSyncPeriodicSyncInterval;

▶️ register() Method

Method

Registers a periodic sync request with the specified tag and options.

Parameters

tag DOMString - Unique identifier
options.minInterval Minimum interval in milliseconds

📋 getTags() / unregister()

Methods

Get all registered periodic sync tags or unregister a specific tag.

getTags() Promise<string[]>
unregister(tag) Promise<void>

⚡ PeriodicSyncEvent Interface

[Exposed=ServiceWorker] interface PeriodicSyncEvent : ExtendableEvent { readonly attribute DOMString tag; }; // Service Worker Event Handler self.addEventListener('periodicsync', (event) => { if (event.tag === 'daily-sync') { event.waitUntil(performDailySync()); } });
