实验性 API

Background Fetch API 是一个实验性功能，允许 Web 应用程序在后台下载大文件（如电影、音频文件和软件），即使网页关闭或浏览器最小化也能继续下载。该 API 需要 Service Worker 支持。

🎯 核心功能

  • ✓ 后台下载大文件（电影、音频、软件等）
  • ✓ 即使网页关闭也能继续下载
  • ✓ 显示下载进度和通知
  • ✓ 支持多文件批量下载
  • ✓ 用户可取消下载操作
  • ✓ 下载完成/失败事件通知

📦 API 组成

BackgroundFetchManager 接口
BackgroundFetchRegistration 接口
BackgroundFetchRecord 接口
BackgroundFetchEvent 事件
BackgroundFetchUpdateUIEvent 事件

💻 快速开始

// 1. 检查 API 支持 if ('BackgroundFetchManager' in self && 'serviceWorker' in navigator) { // 2. 注册 Service Worker const registration = await navigator.serviceWorker.ready; // 3. 启动后台下载 const bgFetch = await registration.backgroundFetch.fetch( 'my-download-id', ['https://example.com/large-file.zip'], { title: '下载大文件', downloadTotal: 100 * 1024 * 1024 // 100MB } ); // 4. 监听进度 bgFetch.addEventListener('progress', () => { console.log(`进度: ${bgFetch.downloaded}/${bgFetch.downloadTotal}`); }); }

📦 BackgroundFetchManager 接口

实验性

BackgroundFetchManager 接口是一个映射，其中键是后台获取 ID，值是 BackgroundFetchRegistration 对象。 通过 ServiceWorkerRegistration.backgroundFetch 属性访问。

接口定义 (Web IDL)

[Exposed=(Window,Worker)] interface BackgroundFetchManager { Promise<BackgroundFetchRegistration> fetch(DOMString id, (RequestInfo or sequence<RequestInfo>) requests, optional BackgroundFetchOptions options = {}); Promise<BackgroundFetchRegistration?> get(DOMString id); Promise<sequence<DOMString>> getIds(); };

▶️ fetch() 方法

方法

启动后台获取操作，给定一个或多个 URL 或 Request 对象。

参数

id DOMString - 唯一标识符
requests RequestInfo | RequestInfo[] - 请求对象
options BackgroundFetchOptions - 配置选项

返回值

Promise<BackgroundFetchRegistration> Promise

🔍 get() 方法

方法

根据 ID 获取已注册的后台获取注册对象。

参数

id DOMString - 后台获取 ID

返回值

Promise<BackgroundFetchRegistration?> Promise

📋 getIds() 方法

方法

返回所有已注册的后台获取 ID 列表。

返回值

Promise<sequence<DOMString>> Promise

⚙️ BackgroundFetchOptions

字典
属性 类型 说明
title DOMString 显示在通知中的标题
icons IconDefinition[] 显示在通知中的图标
downloadTotal unsigned long long 预计下载总大小（字节）

📝 BackgroundFetchRegistration 接口

实验性

BackgroundFetchRegistration 接口表示单个后台获取操作。

接口定义 (Web IDL)

[Exposed=(Window,Worker)] interface BackgroundFetchRegistration : EventTarget { readonly attribute DOMString id; readonly attribute unsigned long long uploadTotal; readonly attribute unsigned long long uploaded; readonly attribute unsigned long long downloadTotal; readonly attribute unsigned long long downloaded; readonly attribute BackgroundFetchResult result; readonly attribute BackgroundFetchFailureReason failureReason; readonly attribute boolean recordsAvailable; attribute EventHandler onprogress; Promise<void> abort(); Promise<BackgroundFetchRecord?> match(Request request, optional CacheQueryOptions options = {}); Promise<sequence<BackgroundFetchRecord>> matchAll(optional Request request, optional CacheQueryOptions options = {}); };

📊 只读属性

属性 类型 说明
id DOMString 后台获取的唯一标识符
uploadTotal unsigned long long 上传总大小（字节）
uploaded unsigned long long 已上传大小（字节）
downloadTotal unsigned long long 下载总大小（字节）
downloaded unsigned long long 已下载大小（字节）
result BackgroundFetchResult 获取结果状态
failureReason BackgroundFetchFailureReason 失败原因
recordsAvailable boolean 是否有可用记录

🔧 方法

abort() Promise<void>

中止后台获取操作。

match(request, options?) Promise<Record?>

匹配单个请求的记录。

matchAll(request?, options?) Promise<Record[]>

匹配所有请求的记录。

📈 BackgroundFetchResult 枚举

枚举
值 说明
"" (空字符串) 获取操作正在进行中
"success" 获取操作成功完成
"failure" 获取操作失败

❌ BackgroundFetchFailureReason 枚举

枚举
值 说明
"" (空字符串) 没有失败
"aborted" 用户或应用中止了操作
"bad-status" 响应状态码不是 OK
"fetch-error" 获取过程中发生错误
"quota-exceeded" 超出存储配额
"download-total-exceeded" 超出预计下载大小

💿 BackgroundFetchRecord 接口

实验性

BackgroundFetchRecord 接口表示单个请求和响应对。

接口定义 (Web IDL)

[Exposed=(Window,Worker)] interface BackgroundFetchRecord { readonly attribute Request request; readonly attribute Promise<Response> responseReady; };

📤 request 属性

只读

返回要获取的资源的请求详情。

类型 Request

📥 responseReady 属性

只读

返回一个 Promise，当响应准备好时解析为 Response 对象。

类型 Promise<Response>

💻 使用示例

// 获取所有记录 const records = await bgFetch.matchAll(); for (const record of records) { // 获取请求信息 console.log('Request URL:', record.request.url); console.log('Request Method:', record.request.method); // 获取响应 const response = await record.responseReady; console.log('Response Status:', response.status); console.log('Response Headers:', response.headers); // 处理响应数据 const blob = await response.blob(); console.log('Data Size:', blob.size); }
ℹ️
Service Worker 事件

Background Fetch API 的事件在 Service Worker 全局作用域中触发。需要在 Service Worker 脚本中监听这些事件。

📡 BackgroundFetchEvent 接口

事件

BackgroundFetchEvent 是传递给 onbackgroundfetchabort 和 onbackgroundfetchclick 事件的事件类型。

[Exposed=ServiceWorker] interface BackgroundFetchEvent : ExtendableEvent { readonly attribute BackgroundFetchRegistration registration; };
属性 类型 说明
registration BackgroundFetchRegistration 触发事件的后台获取注册对象

🎨 BackgroundFetchUpdateUIEvent 接口

事件

BackgroundFetchUpdateUIEvent 是传递给 onbackgroundfetchsuccess 和 onbackgroundfetchfail 事件的事件类型，提供更新 UI 的方法。

[Exposed=ServiceWorker] interface BackgroundFetchUpdateUIEvent : BackgroundFetchEvent { Promise<void> updateUI(optional BackgroundFetchUIOptions options = {}); }; dictionary BackgroundFetchUIOptions { DOMString title; sequence<IconDefinition> icons; };

📋 Service Worker 事件列表

事件 事件类型 触发时机
onbackgroundfetchsuccess BackgroundFetchUpdateUIEvent 后台获取成功完成
onbackgroundfetchfail BackgroundFetchUpdateUIEvent 后台获取失败
onbackgroundfetchabort BackgroundFetchEvent 用户中止后台获取
onbackgroundfetchclick BackgroundFetchEvent 用户点击通知

💻 Service Worker 示例代码

// service-worker.js self.addEventListener('backgroundfetchsuccess', (event) => { console.log('后台获取成功:', event.registration.id); event.waitUntil( event.updateUI({ title: '下载完成!' }) ); }); self.addEventListener('backgroundfetchfail', (event) => { console.log('后台获取失败:', event.registration.failureReason); event.waitUntil( event.updateUI({ title: '下载失败' }) ); }); self.addEventListener('backgroundfetchabort', (event) => { console.log('后台获取已中止:', event.registration.id); }); self.addEventListener('backgroundfetchclick', (event) => { console.log('用户点击了通知'); // 打开应用窗口 event.waitUntil( clients.openWindow('/') ); });

兼容性建议

Background Fetch API 是一个实验性功能，建议在使用前进行特性检测。 对于不支持的浏览器，可以使用传统的 fetch API 配合 Service Worker 的后台同步功能作为替代方案。 确保在 HTTPS 环境下运行，并正确注册 Service Worker。
